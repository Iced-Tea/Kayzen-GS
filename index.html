Responsiveness

Kayzen GS columns are responsive out the box - the following breakpoint classes have been defined to customize the layout - these values are completely flexible and customizable (see Custom Configuration):

  • break-0: 0px,
  • break-1: 460px,
  • break-2: 720px,
  • break-3: 940px,
  • break-4: 1200px

By default, the columns will vertically stack on top of each other when the screen size is less than 940px (break-3). Sometimes you may want different columns to stack at different widths. To override the base value at which columns normally stack, you can add a special helper class to your row.

Sidebar
Article
<div class="row row-break-2">                    
    <div class="span-3">Sidebar</div>
    <div class="span-9">Article</div>
</div>

The above columns will now stack on top of each other when the screen size is less than 720px (break-2).

Sidebar
Article
<div class="row row-break-4">                    
    <div class="span-3">Sidebar</div>
    <div class="span-9">Article</div>
</div>

Likewise, the above columns will now stack on top of each other when the screen size is less than 1200px (break-4).

To cause the columns to not stack at all, you can use the break-0 helper class on your row:

To change the width of a single column manually at specific breakpoints, see Flexible Columns.

Flexible Columns

Magic Columns

Magic Columns are so named as they do not need to have their widths specified - each row of Magic Columns will automatically fill up the row applying an equal width to each column. Magic Columns are perfect for cases where you know each column will always be the same width, or where you need any widow columns to take up any remaining space.

Block Columns

Block Columns have no gutter (so the columns sit flush with each other) and are of equal height. To create a row of Block Columns, add the row-block class to your row.

Responsiveness

By default, the semantic columns are mobile-first. This means they start out at 100% width and respond to their specified values when told, which by default is when the screen size is larger than 940px, as defined by the $col-collapse variable. This value can be overridden by adding the $stack variable to your column.

 
.sidebar {
	@include column(
		$width: (3, 12),
		$stack: breakpoint(break-2) // 720px
	);
}

.content {
	@include column(
		$width: (9, 12),
		$stack: breakpoint(break-2) // 720px
	);
}

The above columns will now stack at the value set for break-2, which is 720px by default.

To create a desktop-first column, you can disable mobile-first like so:

.sidebar {
	@include column(
		$mobile-first: false,
		$width: (3, 12)
	);
}
		
.content {
	@include column(
		$mobile-first: false,
		$width: (9, 12)
	);
}

These columns will start out at their secified width and then have their width forced to 100% to stack on top of each other when told, which by default is when the screen size is less than 940px as defined by the $col-collapse variable.

    @include column(

.portfolio-item {
    @include column(
        $type: flow,
        $respond-to: (

        )
    );

.portfolio-item {
    @include column(
        $type: flow,
        $respond-to: (

.portfolio-item {

    );

Column Aligning

Whilst Kayzen GS does not come with any classes or mixins to align your columns, it is extremely easy to add this functionality yourself. To horizontally align a row of columns, simply set the text-align CSS property of your row to either left, right or center as desired:

This column is centrally aligned.
 
<div class="row" style="text-align: center">
	<div class="span-6">This column is centrally aligned.</div>
</div>

To vertically align your columns, set the vertical-align CSS property of your column. By default, this is set to top.

This column is top aligned.
This column is middle aligned.

<div class="row" style="height: 200px">
    <div class="span-3" style="vertical-align: top">
        This column is top aligned.

        This column is middle aligned.
    </div>
    <div class="span-3" style="vertical-align: bottom">
        This column is bottom aligned.

          This column is middle aligned.
    </div>
</div>
